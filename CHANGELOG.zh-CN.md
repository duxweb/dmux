# 更新日志

项目的重要变更会记录在本文件中。

## [Unreleased]

## [0.5.7] - 2026-04-24

### 修复

- 修复正式版浮动 tooltip 被 SwiftUI overlay 父容器拉宽的问题，让 hover 提示保持紧凑并贴近当前悬停控件。

## [0.5.6] - 2026-04-24

### 修复

- 修复项目 loading 状态稳定性，AI 回复中会一直保持运行态，直到明确收到完成、中断或运行时空闲事件，不再被计时器误清除。
- 修复 Codex 旧 Stop hook 处理，上一轮被打断后的完成事件不会再清掉新一轮 prompt 的 loading 状态。

## [0.5.5] - 2026-04-24

### 修复

- 修复 Codex 记忆提取模型覆盖，按当前 Codex CLI 的 `--model=...` 形式传递设置中的模型，并将内置 Codex 默认模型更新为 `gpt-5.3-codex`。

## [0.5.4] - 2026-04-24

### 修复

- 将全局浮动 tooltip 窗口替换为局部 SwiftUI overlay，修复正式版侧边栏和标题栏 hover 提示整体漂移、呈阶梯状错位的问题。
- 修复 Codex 记忆提取调用，不再用内置默认模型覆盖用户本机 Codex provider 配置。

## [0.5.3] - 2026-04-24

### 修复

- 修复正式版 AI 记忆提取后台 worker 的 CLI 路径解析，现在会从用户登录 shell 环境读取真实 PATH；即使 Codux 从 Finder 启动，也能找到本机安装的 Codex、Claude、Gemini、OpenCode。
- 修复正式版标题栏浮动提示的锚点坐标计算，改为基于真实 AppKit 控件 frame 定位，避免打包后 hover 提示仍然漂移到按钮下方其他位置。

## [0.5.2] - 2026-04-24

### 调整

- 正式 GitHub Release 产物不再单独发布 debug DMG；debug 包仍可通过手动 test-build workflow 生成。

### 修复

- 修复正式版浮动提示定位，将提示锚定到真实控件 overlay，并使用稳定的屏幕坐标显示。
- 修复 AI 记忆提取后台 worker，使 Claude、Codex、Gemini、OpenCode 提供商运行时跳过 Codux 终端 wrapper，直接解析用户本机安装的真实 CLI。

## [0.5.1] - 2026-04-24

### 新增

- 新增记忆提取提供商“自动”模式，会优先使用当前终端工具对应的提供商，然后按提供商优先级回退。

### 修复

- 修复正式版标题栏 hover 提示位置，避免提示气泡下移、无法贴住对应按钮。
- 修复正式版 AI 记忆提取后台 worker 的 CLI 搜索路径，使其和应用内托管终端一样能找到本机 AI CLI。
- 修复记忆提取失败状态，标题栏记忆图标会保持红色并显示最近一次具体失败原因，不再误回到空闲态。
- 修复 Codex 在读取记忆提取 stdin 前退出时可能导致 App 崩溃的问题，现在会转成可恢复的提取失败。
- 修复 Codex 打断旧回复后立刻开始新回复时，旧 stop hook 会短暂清掉左侧项目 loading 状态的问题。
- 优化缺失 CLI 的错误提示，避免把 `/usr/bin/env` 的底层输出误显示为 `.env` 配置问题。

## [0.5.0] - 2026-04-24

### Added

- 新增第一版 AI 记忆系统，支持基于 SQLite 的用户记忆、项目记忆、提取队列、合并后的项目总记忆，以及面向支持工具的少量 working memory 注入。
- 新增 AI 设置页，统一配置 Claude、Codex、Gemini、OpenCode 以及 OpenAI 兼容提取提供商，支持模型、Base URL、API Key 和记忆提取开关。
- 新增标题栏记忆状态指示器，无需进入设置即可看到记忆提取和队列状态。
- 新增项目 `.env` 环境加载能力；文件存在时，Codux 托管终端会自动获得其中的 AI CLI 凭据和代理变量。

### Changed

- 将设置中的 Tools 调整为 AI，并把运行权限、提供商配置和记忆设置合并到统一的 AI 设置界面。
- 外观主题与背景色调整恢复为稳定的“重启后生效”路径，不再对已有终端 surface 做即时应用。
- 更新 README 中的排障日志路径，统一为当前 Codux 支持目录和 `runtime.log` / `runtime.previous.log` 文件名。

### Fixed

- 修复长时间使用后切换项目可能出现的终端焦点漂移问题；现在会忽略其他项目的旧焦点，并在切换项目时清理隐藏终端 responder。
- 修复关闭最后一个可见终端分屏的行为；现在会终止旧会话并立即创建新的项目终端，不再留下不可输入状态或直接拒绝操作。
- 修复长时间运行的 AI 活动续期逻辑，让 hook 驱动的 loading 状态始终绑定真实运行会话，不再提前消失或被旧状态重新带起。
- 修复 Gemini、OpenCode、Codex 在托管终端和记忆提取 worker 中的环境处理，兼容自定义 API Base URL 和凭据配置。

## [0.4.5] - 2026-04-24

### Changed

- 更新内置的 Ghostty 依赖到最新的 AppKit 输入修正版本，让 Codux 直接继承上游终端输入链路修复，不再继续依赖本地兼容性补丁。
- 压缩运行时日志噪音，屏蔽重复的活动状态解析、未变化历史索引、socket 收包以及无变化 hook ingress 明细，只保留状态切换、失败和真正有排障价值的通知诊断日志。

### Fixed

- 修复项目级 AI 活动状态链路，左侧 loading 与 completed 标记现在只由真实的 hook / runtime 会话状态驱动，不会再被 tool-use hook 噪音、项目激活时的陈旧重算或无关的实时会话探测重新带起。
- 修复 Codex 与 Claude 的 hook 事件收敛逻辑，让排队消息、中断后的会话以及 runtime 回填边界在 prompt 提交、完成态和后续新一轮开始之间保持更稳定一致。
- 修复应用托管的 hook 配置清理逻辑，旧的 Codex / Claude tool-use hook 注册现在会从应用自身管理的配置中移除，避免 runtime support 刷新后旧生成项继续带来冗余 hook 流量。

## [0.4.4] - 2026-04-23

### Fixed

- 修复宠物成长计量在新项目首次进入统计时会把历史索引 token 一次性补进经验的问题；现在每个项目都会先建立自己的基线，只有后续新增增长才会计入宠物经验。
- 修复移除或关闭项目后的宠物基线语义；项目被移除时会同步清掉该项目的宠物基线，避免很久之后重新添加并索引时一次性补入大量延迟经验。

## [0.4.2] - 2026-04-23

### Changed

- 重构应用自身运行时路径解析，日志、宠物状态、runtime support 文件以及工具权限设置现在统一落到当前应用自己的 `Application Support` 目录；runtime socket、状态目录等临时产物则统一落到按 owner 隔离的 tmp 根目录。
- 简化调试与运行时日志命名，正式版和开发版不再依赖额外的 `.dev` / `.release` 文件名后缀做区分，版本隔离完全由应用容器目录承担。

### Fixed

- 修复 Codex、Claude、Gemini 的多版本 hook 共存问题，dmux 注入命令现在带有 owner 标识，会保留其他正在使用的应用 owner，同时会自动清理旧版本遗留的无 owner hook 条目和旧 helper 路径注入。
- 修复 Codex 配置注入逻辑，`suppress_unstable_features_warning = true` 现在会作为真正的 TOML 顶层键写入，不会再落进嵌套的 `notice` 表结构里，从而避免启动 warning 持续出现或配置结构被写坏。
- 修复 runtime bootstrap 的临时/持久路径划分，`claude-session-map`、runtime socket 和 agent status 现在都被视为临时运行态产物，不再落入持久 support 目录。
- 修复已有安装的宠物存档迁移问题，应用首次加载时会自动把旧版 `Application Support/dmux*/pet-state.dat` 搬到新的应用私有目录，并按当前运行时命名空间重新加密写回，避免正式版老用户升级后出现宠物档案读不到。
- 修复 Homebrew cask 生成脚本中的清理路径，使 zap 的 `Application Support` 目录与当前正式版真实使用的目录保持一致。

## [0.4.1] - 2026-04-23

### Fixed

- 修复 Codex 运行时配置写入逻辑，`suppress_unstable_features_warning` 现在会写入真正的顶层配置，而不会再污染 `[notice.model_migrations]`，从而避免更新后用户因配置结构损坏而无法启动 Codex。
- 修复实时 AI 会话展示与聚合链路的边界问题，已完成会话现在会继续保留在实时面板中，当前会话 token 卡片始终绑定原始实时 totals，overlay 差值计算不再串入单会话显示路径。
- 修复完成态 baseline、cutoff 之后的 indexed 历史 bucket、损坏的 active-duration 历史行以及托管会话清理等多处统计边界问题，让项目总量、宠物成长输入和实时 overlay 的结果保持更稳定一致。
- 修复正式版运行时 hook/bootstrap 支撑链路，收紧 socket 与配置生成处理，并补充 Codex 配置生成、runtime socket 重连能力以及实时 stats / 会话保留行为的回归测试。

## [0.4.0] - 2026-04-23

### Changed

- 将终端后端切换为 Ghostty，并完成后续工作区接入收口，让分屏、浮窗终端、项目切换以及恢复后的终端会话统一运行在同一条渲染链路上。
- 将应用、终端、AI 统计、Git、设置、历史索引和宠物等多个超大模块继续拆分为更聚焦的小单元，降低后续维护成本和回归风险。
- 调整 AI 历史索引配置与 Ghostty 外观处理，包括内置 Ghostty 主题收口以及后台索引的低占用策略。
- 调整宠物成长、属性评分、多语言属性提示和实时刷新链路，让宠物状态更稳定地跟随领蛋后的实际使用行为，也更容易持续维护。

### Fixed

- 修复 Ghostty 终端在项目切换、浮窗、终端恢复、桥接刷新以及浮出/收回过程中的生命周期问题，避免工作区切换时出现终端内容、输入或滚动异常。
- 修复实时 AI 统计链路中的会话串值和重复计数问题，工具切换、完成态、历史 indexed 叠加以及项目总量显示现在不会再继承旧会话 token、重复叠加 overlay 或在实时/历史之间漂移。
- 修复宠物成长和 AI 统计依赖的历史会话时间过滤语义，跨 cutoff 持续中的会话现在会按正确时间边界统计，而不会整条丢失或混入不该计入的旧 token。
- 修复 Ghostty 内置主题颜色解析问题，内置主题在重启后现在可以正确解析并生效。
- 修复宠物成长账本、水位清理和属性刷新边界问题，孵蛋、经验和属性状态不再因为孤儿实时会话或刷新边界而出现漂移或异常放大。

## [0.3.2] - 2026-04-20

### Fixed

- 修复终端按键透传调整后导致的粘贴问题：`Command` 类编辑快捷键现在继续交给 AppKit 处理，不再被错误地当成原始终端按键转发，从而恢复 `Command+V` 等标准操作。
- 修复在终端中执行 Git 操作后的侧栏自动刷新问题：提交等只改动 `.git` 元数据的操作现在也会立即触发 Git 面板刷新，不再把已提交的更改文件继续错误地留在列表中直到手动刷新。

## [0.3.1] - 2026-04-20

### Changed

- 调整 AI 统计面板切换项目时的呈现方式，先展示轻量摘要，再延后加载较重的详情区块，并将会话历史限制为最近 `20` 条，减少切换时的体感卡顿。
- 调整宠物提醒默认节奏，采用更符合健康建议的初始值：久坐提醒 `30` 分钟、喝水提醒 `2` 小时、深夜提醒 `1` 小时。

### Fixed

- 修复 Codex 和 Claude 在同一会话内连续排队发送消息时的 loading 状态：后续排队消息现在会持续保持 `loading`，直到最后一轮真正完成，不再提前结束或卡住不消失。
- 修复终端按键透传问题，`Shift+Tab` 这类未被应用保留的快捷键现在可以正确传递到底层 AI 终端，不再被外层壳拦截。
- 修复终端上下分屏高度持久化问题，拖动后的 tab 区域高度在切换项目后返回时不再被重置。
- 修复 AI 面板在打开状态下切换项目时的卡顿，通过先同步实时状态、后延迟渲染重内容区块来降低切换阻塞。
- 移除正式版中残留的宠物调试入口及其无用的多语言键，避免发布版本继续暴露内部测试能力。

## [0.3.0] - 2026-04-19

### Added

- 新增桌面宠物系统，包括选蛋领取、孵化、等级与进化成长、传承、按阶段解锁的图鉴，以及配套的精灵资源和进化/孵化特效。
- 新增独立的“设置 > 宠物”标签页，支持宠物启用、静态动画、喝水提醒、久坐提醒和深夜提醒间隔配置。
- 在中英文 README 中补充了宠物系统的用户向说明，并将分屏工作区、Git、AI 用量、每日等级和宠物等功能截图插入到对应介绍位置。

### Changed

- 调整宠物成长规则：宠物蛋领取后成长值从 `0` 开始，后续只根据领取后的 AI 行为持续累积，并改为按小时刷新。
- 重构宠物性格成长算法，移除了智慧成长对工具品牌的直接偏置，将长期 token 成长改为全属性共享底座，并在性格判定中加入均衡门槛，避免轻易被单项锁死。
- 重构共情成长算法，让它更偏向真实的多轮调试、持续修正和反复收敛问题的编程行为，而不再只奖励极短 prompt 的高频刷写。
- 将宠物相关设置从“通用”移入独立的“宠物”标签页，并同步优化图鉴放大层交互和宠物相关文案表现。

### Fixed

- 修复 Claude 完成态处理：`Stop` 现在会直接按 hook 语义标记本轮完成，`Idle` 与 `SessionEnd` 仍负责清理 loading，但不再混淆“真正完成”和“会话收尾”。
- 修复 Codex 在 `Stop` 非 definitive 时偶发卡住 loading 的问题：当 probe 后续确认已 idle 时会将其视为真实完成，同时不再让延迟 stop hook 重新写回过期的 `responding` 状态。
- 修复宠物存储链路，正式版与开发版现在使用独立的本地加密 `.dat` 文件，并去除了 Keychain 弹窗干扰。
- 修复宠物图鉴放大层关闭不稳定的问题，点击任意背景区域现在都能稳定关闭，同时加深遮罩增强聚焦效果。
- 修复深夜宠物提醒时间窗口，将其调整为 `23:00-06:00`，并让提醒触发间隔遵循宠物设置中的配置值。

## [0.2.2] - 2026-04-18

### Changed

- 加固 Claude 包裹启动链路，让 Codux 在启动托管的 Claude 会话时更可靠地解析真实 Claude 二进制，并优先使用系统工具路径。

### Fixed

- 降低终端进程资源耗尽风险：隐藏 pane 不再提前启动 PTY，并为托管 Claude 进程树增加上限，避免其继续吃满用户的进程配额。

## [0.2.1] - 2026-04-18

### Added

- 新增“设置 > 外观”中的终端字号控制，可直接用数字输入调整终端文字大小。
- 新增“设置 > 工具”标签页，用于配置 Codex、Claude Code、Gemini 和 OpenCode 在 Codux 终端中的默认权限模式。
- 新增“设置 > 通知”标签页，可分别为 Bark、ntfy、WxPusher、飞书、钉钉、企微、Telegram、Discord、Slack 和通用 Webhook 配置开关、地址与 token。
- 新增外部通知后台发送链路，已配置的通知渠道会在完成事件后静默异步发送，失败不会阻塞界面，只记录到调试日志。
- 将 WxPusher 收口为 SPT 极简发送模式，移除了无效的 token 输入项，并让设置界面与单参数配置方式保持一致。

### Changed

- 加固 Codex、Claude、Gemini 和 OpenCode 的运行时驱动，让 loading、中断、恢复以及按轮次统计的实时 token 显示都改为由工具自身事件驱动，不再出现跨会话串值。
- 收紧 Codux 终端内的工具二进制解析逻辑，Claude 现在会跟随当前 shell 实际解析到的可执行路径，而不是再去猜测用户的安装位置。
- 调整 AI 统计面板底部状态条，在刷新进行中会隐藏刷新按钮，让更新中状态只保留进度与停止控制。
- 调整应用菜单中的“关于”和“检查更新”项，为两者加入图标并统一归到同一组应用信息入口中。
- 调整通知设置卡片，为不同渠道补充专属字段名、多语言说明文案、对齐更稳定的输入布局，以及跳转到各渠道官方文档的入口。
- 加固外部通知发送链路，统一请求超时与禁用缓存，并补充请求开始、耗时、状态码和脱敏响应摘要等调试日志。

### Fixed

- 修复 Codex、Claude、Gemini 和 OpenCode 的实时用量统计，新会话与恢复历史会话都会从 `0` 开始，只有当前轮回复完成后才显示本轮新增 token。
- 修复工具会话在重开、恢复、中断和多终端场景下的重新绑定逻辑，恢复历史会话时不再继承前一个实时会话的总量、模型或 loading 状态。
- 收紧 live 运行日志，只保留 hook/socket 事件、逻辑会话生命周期、响应状态切换和 token 提交等后续排障真正需要的关键信息。
- 补全了新增加的“工具”设置页多语言文案，并移除了每个权限选择器右侧重复显示的工具名称。
- 修复 Sparkle 更新提示窗口在失去焦点后背景变透明的问题。
- 修复终端分屏重排问题，创建或调整分屏时不再把终端内容挤压成异常的多列文本布局。

## [0.2.0] - 2026-04-17

### Added

- 新增基于 Sparkle + GitHub Releases 的应用内更新链路，包括启动后的后台检查、应用菜单中的检查更新入口、CI 生成签名 `appcast.xml`，以及配套的发布更新文档。

### Fixed

- 修复 Git 文件列表自动刷新问题，本地文件新建、删除以及 AI 生成的改动在面板打开时都会立即反映出来，不再需要手动刷新。
- 修复 Git 面板刷新时丢失选中文件和可见 diff 状态的问题，自动刷新后不再回退到过期或空白详情。
- 修复切换项目、窗口重新激活以及取消最小化后终端焦点恢复问题，Shell 不再需要额外点击一次才能继续输入。
